Типичный рабочий процесс
1.Вы на локальной машине (WSL) изменяете код.
2.Делаете git add, git commit, git push — изменения уходят на GitHub.
3.Через минуту сервер подтягивает их и перезапускает бота.
4.Если вы работаете в WSL и хотите получить изменения, которые кто-то другой запушил на GitHub (или вы сами с сервера), делаете git pull.


Чтобы подтянуть свежие изменения с GitHub на ваш локальный компьютер (WSL), выполните:
1. Простое обновление

cd ~/metanby_tgbot
git pull origin main

Чтобы отправить конкретный файл на GitHub, используйте следующие команды:
1. Добавьте нужный файл
git add имя_файла
git commit -m "Обновлён bot.py"
git push origin main
